# D1 Tables

`equity_symbol_reference` stays reference-only.

Expected columns:

- `nse_symbol`
- `breeze_code`
- `nse_company_name`
- `listing_date`
- `isin`

`market_data_sync_state` stores sync state.

Columns:

- `market_type`: `cash`, `futures`, or `options`.
- `nse_symbol`: NSE symbol.
- `status`: `in_progress`, `completed`, or `failed`.
- `from_date`: Synced range start.
- `to_date`: Synced range end.
- `row_count`: Rows staged locally and uploaded.
- `local_path`: Local staging root used for the run.
- `r2_prefix`: R2 prefix used for the run.
- `error`: Last failure message.
- `started_at`: Last start timestamp.
- `completed_at`: Completion timestamp.
- `updated_at`: Last state update timestamp.

Apply migration:

```bash
wrangler d1 execute "$D1_DATABASE_ID" --remote --file migrations/d1/001_market_data_sync_state.sql
```

Pending cash query behavior:

- Pick first `equity_symbol_reference` row.
- Skip rows where `market_data_sync_state.market_type = 'cash'`, `status = 'completed'`, and `to_date` is current for the requested run.
- Re-sync from `to_date + 1` when older completed data exists.
- Failed rows remain eligible for retry.
